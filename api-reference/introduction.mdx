## Status dan error

API mengembalikan kode status HTTP standar:

- `200` untuk request sukses
- `201` untuk resource baru
- `400` untuk payload tidak valid
- `401` untuk token tidak valid atau tidak ada
- `404` untuk resource tidak ditemukan
- `422` untuk validasi bisnis gagal
- `429` untuk rate limit

<Note>
  Simpan `Idempotency-Key` untuk operasi create yang berhubungan dengan pembayaran agar request ganda tidak membuat duplikasi transaksi.
</Note>
